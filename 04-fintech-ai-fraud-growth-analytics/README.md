# Fintech AI Fraud & Growth Analytics

## Industry
Fintech / Financial Services

## Research Question
Can AI identify financially vulnerable customers and fraudulent transactions early enough to prevent harmful debt spirals — and enable smarter intervention before default?

## Project Goal
Analyze financial transaction data to build AI-assisted risk scoring that identifies fraudulent patterns, evaluates anomaly indicators, and recommends intervention priorities to reduce fraud losses while minimizing false positives.

## Tools Used
- Excel
- SQL
- Python
- Tableau
- AI-assisted analysis

## Dataset
**Source:** Fraud Detection Dataset — Kaggle (free)
**Files:** 10 CSVs covering transactions, customers, fraud indicators, anomaly scores, merchants

## Repository Structure
- `data` — raw, cleaned, and sample datasets
- `excel` — workbook for KPI tracking and fraud simulation
- `sql` — schema and analytical queries
- `python` — scripts for cleaning, EDA, and AI-assisted risk scoring
- `dashboard` — Tableau dashboard plan and publishing notes
- `ai-research` — prompts, research framing, and AI-use disclosure
- `outputs` — summary tables and recommendations
- `screenshots` — charts and process visuals for GitHub
- `notes` — workflow and upload instructions

## Key Questions Answered
1. Which transaction categories have the highest fraud rates?
2. How effective is AI risk scoring at capturing actual fraud?
3. What hourly patterns exist in fraudulent transactions?
4. How should intervention priorities be assigned?
5. Can AI reduce false positives while maintaining detection coverage?

## Key Findings
- **Overall fraud rate:** 4.5%
- **AI High Risk tier:** Concentrates the most dangerous transactions
- **Category analysis:** Fraud rates vary across Food, Retail, Travel, Online, Other
- **Hourly patterns:** Time-based fraud variation detected
- **AI intervention:** Blended scoring enables efficient triage
